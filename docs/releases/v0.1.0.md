# v0.1.0

Initial release of omni-openrouter, a thick provider for omnillm-core that enables access to 300+ AI models through OpenRouter's unified API.

## Features

### OpenRouter Provider

- Full chat completion support (streaming and non-streaming)
- Tool/function calling with auto, none, required, and specific tool modes
- Vision support for multimodal models
- JSON mode for structured output
- Message role handling for system, user, assistant, and tool messages
- Error wrapping with omnillm-core error types
- Automatic provider registration via package import

### OAuth PKCE Authentication

- Secure OAuth 2.0 PKCE flow for interactive authentication
- Local HTTP callback server on random available port
- Browser-based authentication with cross-platform support (macOS, Linux, Windows)
- Credential storage via omnivault
- 5-minute authentication timeout

## Installation

```bash
go get github.com/plexusone/omni-openrouter@v0.1.0
```

## Usage

### As omnillm Provider

```go
import _ "github.com/plexusone/omni-openrouter/omnillm"

client, err := omnillm.NewChatClient(omnillm.Config{
    Provider: "openrouter",
    APIKey:   os.Getenv("OPENROUTER_API_KEY"),
    Model:    "anthropic/claude-3.5-sonnet",
})
```

### OAuth Authentication

```go
import "github.com/plexusone/omni-openrouter/auth"

apiKey, err := auth.Login(context.Background())
```
